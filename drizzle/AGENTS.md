# AGENTS.md

## Scope

This folder contains Drizzle migrations and migration metadata.

## Rules

- Treat SQL migrations as append-only after they are applied.
- Prefer generating migrations from `src/db/schema.ts` with `npm run db:generate`.
- Do not edit `_journal.json` or snapshot files unless repairing migration metadata intentionally.
- Keep migration filenames generated by Drizzle.
- Review migrations before applying, especially enum changes, unique indexes, and destructive alterations.

## Verification

- Apply migrations with `npm run db:migrate`.
- Run `npm run typecheck` after schema changes.
- Run `npm run build` when route or server behavior depends on the new schema.

## Production Safety

- Avoid destructive schema changes without a data migration/rollback plan.
- For NOT NULL columns on existing tables, provide safe defaults or staged migrations.
- Add indexes concurrently only if production scale requires it and the deployment process supports it.
